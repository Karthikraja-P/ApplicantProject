### Q23: John needs 13 bottles of water from the store. John can only carry 3 at a time. What is the minimum number of trips John needs to make?
**Options**:
- Option 1: 3
- Option 2: 4
- Option 3: 4 1/2
- Option 4: 5
- Option 5: 6

**Correct Answer**: Option 4
**Explanation**: 13 divided by 3 is 4 remainder 1. So 4 trips is not enough; he needs a 5th trip. Answer: 5.

## 💻 Technical Test: Machine Learning

### Q1: You train and evaluate a model on the same dataset. The Sharpe looks extraordinary. What is the fundamental problem?
**Category**: ML Foundations
Option 1: Sharpe requires 252+ trading days — shorter samples make it statistically unreliable
Option 2: In-sample evaluation cannot detect overfitting — the model memorised the data rather than learning generalisable patterns
Option 3: The model must be retrained monthly — a single training run decays as markets change
Option 4: Sharpe is not an ML metric — use accuracy or F1 instead

**Correct Answer**: Option 2

---
### Q2: Training loss decreases over 200 epochs. Validation loss stops improving after epoch 40 and starts rising. What is the correct action?
**Category**: ML Foundations
Option 1: Train to epoch 200 — early validation fluctuations are noise; the final weights are most informed
Option 2: Increase the learning rate — the rising validation loss indicates the optimiser is stuck in a local minimum
Option 3: Stop at the epoch with the lowest validation loss — continuing only deepens overfitting
Option 4: Switch to a simpler model — rising validation loss always means the architecture is too large

**Correct Answer**: Option 3

---
### Q3: You split time-series trade data using standard k-fold, randomly shuffling rows first. What is the critical flaw?
**Category**: ML Foundations
Option 1: K-fold doesn't stratify by class — imbalanced trade outcomes will be unevenly distributed across folds
Option 2: Random shuffling destroys temporal order — training folds inadvertently contain future observations, inflating all performance estimates
Option 3: K-fold underestimates variance because each fold sees the full feature distribution
Option 4: K-fold assumes Gaussian labels — financial returns are fat-tailed, making fold statistics unreliable

**Correct Answer**: Option 2

---
### Q4: You run 50 feature-selection tests at p < 0.05. Eight features pass. You report them as significant. What is the flaw?
**Category**: ML Foundations
Option 1: p < 0.05 is too conservative for financial data — the industry standard is 0.10
Option 2: Sequential testing creates correlated subsets — the 8 features share information that should be penalised jointly
Option 3: 50 tests at p < 0.05 expects ~2.5 false discoveries by chance — without multiple-testing correction, spurious features likely passed
Option 4: Significance confirms information content, not out-of-sample usefulness — the wrong criterion entirely

**Correct Answer**: Option 3

---
### Q5: You apply L1 regularisation to a linear return-prediction model. What is the key practical difference from L2?
**Category**: ML Foundations
Option 1: L1 penalises large weights more aggressively, making the model conservative near extreme returns
Option 2: L2 penalises the learning rate directly; L1 penalises weight magnitudes — they are effectively equivalent in practice
Option 3: L1 drives irrelevant weights exactly to zero, acting as embedded feature selection — L2 only shrinks weights
Option 4: L1 is only appropriate for classifiers — for continuous return regression, L2 is always preferred

**Correct Answer**: Option 3

---
### Q6: SHAP values explain a GBT model. For one specific trade, RSI_14 has a large positive SHAP value. What does this mean precisely?
**Category**: ML Foundations
Option 1: RSI_14 has the largest positive weight in the model's weight vector for this prediction
